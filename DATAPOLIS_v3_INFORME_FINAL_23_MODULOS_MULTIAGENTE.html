📐 Estándares de APIs

  • REST JSON con versionado (/api/v1/)
  • OpenAPI 3.1 para documentación
  • FAPI 2.0 para Open Finance
  • ISO 20022 para mensajería financiera
  • GeoJSON para datos espaciales

🔐 Seguridad y Permisos

  • JWT con refresh tokens
  • OAuth2 / OIDC
  • mTLS para Open Finance
  • RBAC (Role-Based Access Control)
  • Multi-tenant isolation

📊 Observabilidad

  • Prometheus + Grafana
  • Structured logging (JSON)
  • Distributed tracing (Jaeger)
  • Error tracking (Sentry)
  • Audit trail completo

💰 4. MÓDULOS FINTECH (6)

📌 Vertical FinTech: Servicios financieros especializados en real estate: Open Finance, scoring, valorización, liquidaciones y cumplimiento tributario.

M01-OF Open Finance NCG 514 (FAPI 2.0)

📋 Propósito

Implementación completa del Sistema de Finanzas Abiertas de Chile (NCG 514 CMF). Permite a DATAPOLIS actuar como TPP (Third Party Provider) para acceder a datos financieros con consentimiento.

⚡ Funcionalidades Clave

  • Flujo OAuth2 + FAPI 2.0 completo
  • Gestión de consentimientos
  • Acceso a cuentas y saldos
  • Historial de transacciones
  • Iniciación de pagos (PIS)
  • Directorio CMF de participantes

🔐 Seguridad FAPI 2.0

  • mTLS obligatorio
  • Certificados X.509
  • Tokens JWT firmados
  • PAR (Pushed Authorization)

🔗 Normativa

NCG 514, Ley 21.521, ISO 20022, FAPI 2.0

📅 Deadline

Abril 2026 (obligatorio)

M03 Credit Score ML 5 Dimensiones

📋 Propósito

Scoring crediticio propietario basado en 5 dimensiones: capacidad de pago, historial crediticio, colateral, capital y condiciones de mercado.

🧮 Dimensiones del Score

  • D1: Capacidad de pago (30%)
  • D2: Historial crediticio (25%)
  • D3: Colateral (20%)
  • D4: Capital (15%)
  • D5: Condiciones (10%)

🤖 Modelo ML

  • XGBoost + SHAP explainability
  • Reentrenamiento mensual
  • AUC-ROC > 0.85

📤 Salidas

  • Score 0-1000
  • Clasificación (AAA a D)
  • SHAP feature importance

🔗 Normativa

NCG 440, Basel III/IV

M04 Valorización IVS 2022

📋 Propósito
